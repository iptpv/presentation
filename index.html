MCSS: от теории к практике

Почему MCSS?

  1. БЭМ
    • Нет правил взаимодействия модулей-слоев
    • Сложно использовать без специнструментов
  2. SMACSS
    • Базовый слой содержит только layout
    • Не достаточно изоляции блоков

Компоновка модулей: фундамент

Компоновка модулей: фундамент

  • Reset.css, normalaize.css...
  • Малый вес селекторов

Компоновка модулей: базовый

Компоновка модулей: базовый

  • Формы, кнопки, таблицы...
  • Может переопределять только фунтамент

Компоновка модулей: проектный

Компоновка модулей: проектный

  • Формa логина, верхнее меню...
  • Может переопределять фундамент и базовый слой

Компоновка модулей: косметический

Компоновка модулей: косметический

  • Имеет наибольший вес
  • Не зависит от других слоев
  • Глобальные модификаторы
  • Может отсутствовать

Контекст

Стили располагаются рядом с модифицируемыми классами

  • media queries
  • специфичные для браузера свойства

Пример

				<class="menu _boxing header-menu__i right_zero">
				 <a class="header-menu__i__link" href="/">
				 </a>
				</div>

Javascript

  • Js-модуль находится рядом с файлами стилей которые использует

  • БезDOMные js-файлы находятся в слое фундамента и могут не иметь файлов стилей (библиотека для работы с cookie, jquery...)
